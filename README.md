## 1. HTML基本構造タグ

### `<!DOCTYPE html>`
- **意味**: これはHTML5のファイルですよ、とブラウザに伝える宣言
- **用途**: 必ずHTMLファイルの一番最初に書く。これがないとブラウザが古いモードで動くことがある
- **ポートフォリオでの使用箇所**: 1行目

```html
<!DOCTYPE html>
```

---

### `<html lang="ja">`
- **意味**: HTMLファイル全体を囲む一番外側のタグ。`lang="ja"` は「このページは日本語です」という設定
- **用途**: ページ全体のラッパー。langは検索エンジンや音声読み上げソフトが言語を判定するのに使う
- **ポートフォリオでの使用箇所**: 2行目

---

### `<head>`
- **意味**: ページに表示されない「設定情報」を入れる場所
- **用途**: タイトル・フォント読み込み・CSSなど、見えない設定をまとめる
- **ポートフォリオでの使用箇所**: 3〜486行目

---

### `<meta charset="UTF-8">`
- **意味**: 文字コードをUTF-8に設定する
- **用途**: 日本語が文字化けしないようにするために必須。必ず `<head>` の中に書く
- **ポートフォリオでの使用箇所**: 4行目

---

### `<meta name="viewport" ...>`
- **意味**: スマホなどのモバイル端末でページがちゃんと表示されるよう調整する設定
- **用途**: `width=device-width` で画面幅に合わせる。`initial-scale=1.0` で初期ズームを等倍に
- **ポートフォリオでの使用箇所**: 5行目

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

---

### `<title>`
- **意味**: ブラウザのタブに表示されるページのタイトル
- **用途**: SEO（検索エンジン対策）にも影響する。`<head>` の中に書く
- **ポートフォリオでの使用箇所**: 6行目

---

### `<link>`
- **意味**: 外部ファイルをHTMLに読み込む
- **用途**: Googleフォントや外部CSSを読み込むときに使う
- **ポートフォリオでの使用箇所**: 7行目（Googleフォントの読み込み）

```html
<link href="フォントのURL" rel="stylesheet">
```

---

### `<style>`
- **意味**: HTMLファイルの中にCSSを直接書くためのタグ
- **用途**: CSSファイルを別に作らず、HTMLの中にまとめてデザインを書くときに使う
- **ポートフォリオでの使用箇所**: 8〜485行目

---

### `<body>`
- **意味**: ブラウザに実際に表示される内容を入れる場所
- **用途**: テキスト・画像・ボタンなど、見える要素はすべてここに書く
- **ポートフォリオでの使用箇所**: 487行目〜

---

### `<script>`
- **意味**: JavaScriptのコードを書くタグ
- **用途**: ページに動きをつけるJSコードをここに書く。通常は `</body>` の直前に置く
- **ポートフォリオでの使用箇所**: 643〜653行目（スクロールアニメーション）

---

## 2. レイアウト・グループ化タグ

### `<div>`
- **意味**: 意味を持たないただの「箱」。複数の要素をまとめるために使う
- **用途**: レイアウトを組む・CSSでスタイルをつけるグループを作る
- **ポートフォリオでの使用箇所**: `.hero`・`.avatar`・`.hero-tags` など多数

```html
<div class="hero">
  <!-- ここにまとめたい要素を入れる -->
</div>
```

---

### `<nav>`
- **意味**: ナビゲーション（メニュー）を表す意味付きタグ
- **用途**: サイトのメニューリンクをまとめる場所。`<div>` と動作は同じだが意味が明確
- **ポートフォリオでの使用箇所**: 490〜498行目

---

### `<section>`
- **意味**: ページの一つのまとまった「セクション（区画）」を表す
- **用途**: About・スキル・実績など、内容のかたまりごとに使う
- **ポートフォリオでの使用箇所**: `#about` `#skills` `#works` `#contact`

---

### `<footer>`
- **意味**: ページの一番下のフッター部分を表す
- **用途**: 著作権表示・リンク集・連絡先などをまとめる
- **ポートフォリオでの使用箇所**: 632〜641行目

---

### `<ul>` / `<li>`
- **意味**: `<ul>` は箇条書きリスト、`<li>` はリストの各項目
- **用途**: ナビリンクのリストなど。CSSで点（bullet）を消してメニューとして使うことが多い
- **ポートフォリオでの使用箇所**: ナビゲーションの `<ul class="nav-links">`

```html
<ul class="nav-links">
  <li><a href="#about">about</a></li>
</ul>
```

---

## 3. テキスト系タグ

### `<h1>` / `<h2>` / `<h3>`
- **意味**: 見出しタグ。数字が小さいほど重要度が高い（h1が最大）
- **用途**: `<h1>` はページに1つだけ使うメインタイトル。`<h2>` はセクションタイトル
- **ポートフォリオでの使用箇所**: `<h1>張　帆</h1>`・各セクションの `<h2>`

---

### `<p>`
- **意味**: 段落（paragraph）。テキストのかたまりを表す
- **用途**: 本文・説明文・サブテキストなどに使う
- **ポートフォリオでの使用箇所**: 自己紹介文・ラベルテキストなど多数

---

### `<span>`
- **意味**: `<div>` のインライン版。テキストの一部だけをグループ化する
- **用途**: テキストの一部だけ色を変える・バッジ（タグ）を作るなど
- **ポートフォリオでの使用箇所**: `.badge`・`.skill-name`・`.work-tag` など

```html
<span class="badge badge-purple">個人事業主</span>
```

---

## 4. リンク・メディア系タグ

### `<a>`
- **意味**: リンク（アンカー）タグ。クリックすると別の場所に飛ぶ
- **用途**: 別ページへのリンク・同じページ内のジャンプ・メールリンクなどに使う
- **ポートフォリオでの使用箇所**: ナビリンク・メールボタン・フッターリンク

```html
<!-- 別ページへ -->
<a href="https://example.com">リンクテキスト</a>

<!-- 同じページ内ジャンプ -->
<a href="#about">aboutへ</a>

<!-- メールを開く -->
<a href="mailto:your@email.com">メールを送る</a>
```

---

## 5. HTML属性

### `id="..."`
- **意味**: その要素に固有の名前（ID）をつける。ページ内で同じIDは1つだけ
- **用途**: CSS・JS・ページ内リンクから特定の要素を指定するために使う
- **ポートフォリオでの使用箇所**: `id="about"` `id="skills"` `id="works"` `id="contact"`

```html
<section id="about">
```
```css
/* CSSでは # でIDを指定 */
#about { ... }
```

---

### `class="..."`
- **意味**: 要素にクラス名（グループ名）をつける。同じクラスを複数の要素に使える
- **用途**: CSSでまとめてスタイルをつける・JSで要素を操作するために使う
- **ポートフォリオでの使用箇所**: `.hero` `.skill-card` `.fade-up` など多数

```html
<div class="skill-card fade-up">
```
```css
/* CSSでは . でクラスを指定 */
.skill-card { ... }
```

---

### `href="..."`
- **意味**: `<a>` タグのリンク先URLを指定する属性
- **用途**: クリックしたときにどこへ飛ぶかを指定する
- **ポートフォリオでの使用箇所**: ナビリンク・メールボタン

| 書き方 | 意味 |
|---|---|
| `href="https://..."` | 外部サイトへ |
| `href="#about"` | 同じページの `id="about"` へジャンプ |
| `href="mailto:xxx"` | メールアプリを開く |

---

### `rel="stylesheet"`
- **意味**: `<link>` タグで読み込むファイルがCSSであることを示す
- **用途**: 外部CSSやGoogleフォントを読み込むときに必ずつける
- **ポートフォリオでの使用箇所**: 7行目のGoogleフォント読み込み

---

### `style="..."`
- **意味**: HTMLタグに直接CSSを書く属性（インラインスタイル）
- **用途**: 一つの要素だけに個別のスタイルを当てるとき。多用は避けた方がよい
- **ポートフォリオでの使用箇所**: `style="width:55%"` などスキルバーの幅指定

---

## 6. CSSレイアウト・配置

### `display: flex`
- **意味**: 要素を横並びにするレイアウトモード（Flexbox）
- **用途**: ナビのメニューを横並び・カードを並べる・中央揃えなど非常によく使う
- **ポートフォリオでの使用箇所**: ナビ・アバター・スキルカードなど多数

```css
.nav-links {
  display: flex;
  gap: 28px; /* 要素の間隔 */
}
```

---

### `display: grid`
- **意味**: 格子状（グリッド）にレイアウトするモード
- **用途**: 2列・3列のカード配置など、格子状の整ったレイアウトに使う
- **ポートフォリオでの使用箇所**: `.skill-grid`（2列のスキルカード）

```css
.skill-grid {
  display: grid;
  grid-template-columns: repeat(2, 1fr); /* 2列、均等幅 */
  gap: 12px;
}
```

---

### `flex-direction: column`
- **意味**: flexの並び方を縦（上から下）にする
- **用途**: デフォルトは横並び（row）。縦に積み重ねたいときに使う
- **ポートフォリオでの使用箇所**: `.works-list`

---

### `justify-content`
- **意味**: flex/gridの主軸方向（横）の揃え方を指定
- **用途**: 要素をどう配置するかを決める

| 値 | 意味 |
|---|---|
| `flex-start` | 左寄せ（デフォルト） |
| `flex-end` | 右寄せ |
| `center` | 中央 |
| `space-between` | 両端に要素、残りを均等に |

- **ポートフォリオでの使用箇所**: ナビ（`space-between`）・バッジ（`center`）

---

### `align-items`
- **意味**: flex/gridの交差軸方向（縦）の揃え方を指定
- **用途**: 横並び要素を上下中央に揃えるなど

| 値 | 意味 |
|---|---|
| `flex-start` | 上揃え |
| `center` | 上下中央 |
| `flex-end` | 下揃え |

- **ポートフォリオでの使用箇所**: ナビ・アバター・ワークカードなど

---

### `position`
- **意味**: 要素の配置方法を指定する

| 値 | 意味 |
|---|---|
| `static` | デフォルト。通常の流れ |
| `relative` | 通常位置を基準に `top` `left` などでずらせる |
| `absolute` | 親要素を基準に好きな場所に置ける |
| `sticky` | スクロールしても一定位置に固定される |

- **ポートフォリオでの使用箇所**: ナビ（`sticky`）・装飾円（`absolute`）

---

### `z-index`
- **意味**: 要素の重なり順序を数字で指定。数字が大きいほど前面に来る
- **用途**: 複数の要素が重なるとき、どれを上に表示するか制御する
- **ポートフォリオでの使用箇所**: ナビ（`z-index: 100`）・ヒーロー内テキスト（`z-index: 1`）

---

### `overflow: hidden`
- **意味**: 要素からはみ出たコンテンツを隠す
- **用途**: 装飾の円などが親要素の外にはみ出ないようにする
- **ポートフォリオでの使用箇所**: `.hero`・`.contact-card`

---

### `flex-wrap: wrap`
- **意味**: flexの要素が一行に収まらないとき、折り返して次の行に並べる
- **用途**: バッジなどが画面に収まらないとき自動で折り返す
- **ポートフォリオでの使用箇所**: `.hero-tags`・`.work-tags`

---

### `flex-shrink: 0`
- **意味**: flex内でその要素だけ縮まないようにする
- **用途**: アイコンなどのサイズが縮んでほしくない要素に使う
- **ポートフォリオでの使用箇所**: `.work-icon`

---

### `gap`
- **意味**: flex/grid要素の間隔を指定
- **用途**: 要素同士の隙間を作る。`margin` より簡単に使える
- **ポートフォリオでの使用箇所**: ナビ・スキルグリッド・バッジなど多数

---

### `@media (max-width: 480px)`
- **意味**: 画面幅が480px以下のとき（スマホ）だけ適用するCSSを書くブロック
- **用途**: スマホでレイアウトを変えるレスポンシブデザインに使う
- **ポートフォリオでの使用箇所**: ファイル末尾のレスポンシブ設定

```css
@media (max-width: 480px) {
  .hero h1 { font-size: 28px; } /* スマホでは文字を小さく */
}
```

---

## 7. CSSサイズ・余白

### `margin`
- **意味**: 要素の「外側」の余白
- **用途**: 要素と要素の間に隙間を作る

```css
margin: 上 右 下 左;   /* 4方向個別 */
margin: 上下 左右;      /* 2値 */
margin: 0 auto;        /* 左右中央寄せ */
```

- **ポートフォリオでの使用箇所**: `margin: 0 auto`（セクションの中央寄せ）など多数

---

### `padding`
- **意味**: 要素の「内側」の余白
- **用途**: 要素の端とテキストの間に隙間を作る

```css
padding: 上下 左右;
padding: 48px 24px;  /* 上下48px、左右24px */
```

- **ポートフォリオでの使用箇所**: セクション・カードなど多数

---

### `width` / `height`
- **意味**: 要素の横幅・縦幅
- **用途**: 要素のサイズを固定値（px）や割合（%）で指定する
- **ポートフォリオでの使用箇所**: `.avatar`（96px）・スキルバーの幅（%）など

---

### `max-width`
- **意味**: 要素の最大横幅。これ以上広くならない
- **用途**: コンテンツが画面いっぱいに広がりすぎないよう制限する
- **ポートフォリオでの使用箇所**: `section { max-width: 680px; }`

---

### `border-radius`
- **意味**: 要素の角を丸くする
- **用途**: カードの角丸・丸いアバター・ピル型バッジなど

```css
border-radius: 8px;    /* 少し丸い */
border-radius: 50%;    /* 完全な円 */
border-radius: 99px;   /* ピル型（横長の丸） */
```

- **ポートフォリオでの使用箇所**: アバター・カード・バッジなど多数

---

## 8. CSS色・背景・見た目

### `background` / `background-color`
- **意味**: 要素の背景色や背景画像を設定する
- **用途**: カードの白背景・ヒーローのパステル背景など
- **ポートフォリオでの使用箇所**: 全体に多数

```css
background: white;
background: #EEF2FF;           /* 16進数カラーコード */
background: rgba(255,255,255,0.9); /* 半透明の白 */
```

---

### `rgba(...)`
- **意味**: 赤・緑・青・透明度（0〜1）で色を指定する
- **用途**: 半透明の色を作るとき。背景を薄く重ねるなど
- **ポートフォリオでの使用箇所**: ナビの半透明背景・装飾の薄い円

```css
rgba(255, 255, 255, 0.9)  /* 白を90%の不透明度で */
rgba(0, 0, 0, 0.08)       /* 黒を8%だけ（薄い影） */
```

---

### `opacity`
- **意味**: 要素全体の透明度（0=完全透明、1=完全不透明）
- **用途**: 装飾要素を薄く表示する・フェードアニメーション
- **ポートフォリオでの使用箇所**: ヒーローの装飾円・fade-upの初期状態

---

### `border`
- **意味**: 要素の枠線を設定する
- **用途**: カードの枠線・区切り線など

```css
border: 1px solid #E5E7EB;  /* 太さ スタイル 色 */
border-bottom: 1px solid var(--border); /* 下だけ */
border: 2px dashed #ccc;    /* 破線 */
```

- **ポートフォリオでの使用箇所**: カード・QRプレースホルダーなど

---

### `box-shadow`
- **意味**: 要素に影（シャドウ）をつける
- **用途**: カードが浮いているように見せるホバー時のエフェクト

```css
box-shadow: 横ずれ 縦ずれ ぼかし 色;
box-shadow: 0 8px 24px rgba(0,0,0,0.08);
```

- **ポートフォリオでの使用箇所**: `.skill-card:hover`・`.avatar`

---

### `backdrop-filter: blur(...)`
- **意味**: 要素の背後をぼかす効果
- **用途**: すりガラスのような半透明ナビゲーションを作る
- **ポートフォリオでの使用箇所**: スクロールしても固定されるナビ

---

### `linear-gradient(...)`
- **意味**: 2色以上をグラデーションでつなぐ背景
- **用途**: スキルバーのカラフルな色など

```css
background: linear-gradient(90deg, #4F46E5, #818CF8);
/* 方向, 色1, 色2 */
```

- **ポートフォリオでの使用箇所**: `.skill-bar`

---

## 9. CSSテキスト・フォント

### `font-family`
- **意味**: テキストのフォント（書体）を指定する
- **用途**: 日本語フォント・英語フォントを使い分ける
- **ポートフォリオでの使用箇所**: body全体・ナビロゴ・フッター

```css
font-family: 'Noto Sans JP', sans-serif;
/* 第1候補, フォールバック(なければゴシック体) */
```

---

### `font-size`
- **意味**: 文字の大きさ
- **用途**: 見出し・本文・ラベルで大きさを使い分ける
- **ポートフォリオでの使用箇所**: 全体に多数。`px` で指定

---

### `font-weight`
- **意味**: 文字の太さ
- **用途**: 見出しを太く・本文は普通など

| 値 | 意味 |
|---|---|
| `400` | 普通（normal） |
| `500` | 少し太め |
| `700` | 太字（bold） |

---

### `color`
- **意味**: 文字の色
- **用途**: メインテキスト・サブテキスト・アクセントカラーなど
- **ポートフォリオでの使用箇所**: 全体に多数

---

### `text-decoration`
- **意味**: テキストの装飾（下線など）
- **用途**: リンクの下線を消すときによく使う

```css
text-decoration: none;      /* 下線なし */
text-decoration: underline; /* 下線あり */
```

- **ポートフォリオでの使用箇所**: `<a>` タグの下線を消すのに使用

---

### `text-align`
- **意味**: テキストの横方向の揃え方
- **用途**: 中央揃え・右揃えなど

```css
text-align: center; /* 中央 */
text-align: right;  /* 右揃え */
text-align: left;   /* 左揃え（デフォルト） */
```

- **ポートフォリオでの使用箇所**: ヒーロー・フッター・contactカード

---

### `letter-spacing`
- **意味**: 文字と文字の間隔
- **用途**: タイトルを広げて高級感・ラベルを大文字で広げて整える
- **ポートフォリオでの使用箇所**: `h1`・セクションラベル・フッター名前

---

### `line-height`
- **意味**: 行の高さ（行間）
- **用途**: 本文の読みやすさを調整する。1.6〜1.8が読みやすい値
- **ポートフォリオでの使用箇所**: `body { line-height: 1.7; }`

---

### `text-transform: uppercase`
- **意味**: テキストを大文字に変換して表示する
- **用途**: セクションラベルをすべて大文字にしてデザイン的に見せる
- **ポートフォリオでの使用箇所**: `.section-label`（`about me` → `ABOUT ME`）

---

### `list-style: none`
- **意味**: リスト（`<ul>`）の先頭の点（●）を消す
- **用途**: ナビメニューをリストで作るとき、点が不要なので消す
- **ポートフォリオでの使用箇所**: `.nav-links`

---

## 10. CSSアニメーション・動き

### `transition`
- **意味**: プロパティが変化するとき、その変化をなめらかにアニメーションする
- **用途**: ホバー時に色が急に変わらず、ふわっと変わるようにする

```css
transition: color 0.2s;           /* colorが0.2秒でゆっくり変わる */
transition: transform 0.2s, box-shadow 0.2s; /* 複数指定 */
```

- **ポートフォリオでの使用箇所**: ナビリンク・カードホバー・スクロールアニメーション

---

### `transform`
- **意味**: 要素を移動・拡大縮小・回転させる（実際の位置は変わらない）
- **用途**: ホバーで浮き上がる・スクロールアニメーションでY方向に移動など

```css
transform: translateY(-2px);  /* 上に2px移動 */
transform: translateY(32px);  /* 下に32px移動 */
transform: scale(1.04);       /* 1.04倍に拡大 */
```

- **ポートフォリオでの使用箇所**: カードホバー・ボタンホバー・fade-upアニメーション

---

### `@keyframes`
- **意味**: アニメーションの動きを定義するブロック。`from`（始め）→ `to`（終わり）で変化を書く
- **用途**: ページロード時のフェードインアニメーションなど

```css
@keyframes fadeUp {
  from { opacity: 0; transform: translateY(20px); } /* 始まり */
  to   { opacity: 1; transform: translateY(0); }    /* 終わり */
}
```

- **ポートフォリオでの使用箇所**: ヒーローの要素が下から出てくるアニメーション

---

### `animation`
- **意味**: `@keyframes` で定義したアニメーションを要素に適用する
- **用途**: ページロード時に要素を動かす

```css
animation: fadeUp 0.6s ease both;
/* アニメーション名 時間 緩急 終了後の状態 */
```

---

### `animation-delay`
- **意味**: アニメーション開始を遅らせる
- **用途**: 複数要素を順番に（時間差で）出現させてリズムを作る
- **ポートフォリオでの使用箇所**: ヒーローの各要素（0.1秒ずつずらす）

---

### `scroll-behavior: smooth`
- **意味**: ページ内リンクのジャンプをなめらかにスクロールする
- **用途**: ナビのリンクをクリックしたときにふわっとスクロール
- **ポートフォリオでの使用箇所**: `html { scroll-behavior: smooth; }`

---

## 11. CSS変数（カスタムプロパティ）

### `:root { --変数名: 値; }`
- **意味**: CSS全体で使いまわせる変数を定義する場所
- **用途**: 色やサイズを一箇所で管理し、変更が楽になる

```css
:root {
  --primary: #4F46E5;
}
```

---

### `var(--変数名)`
- **意味**: `:root` で定義したCSS変数を使う
- **用途**: `color: var(--primary)` のように書くと、後で色を変えたいとき `:root` だけ修正すればOK
- **ポートフォリオでの使用箇所**: 全体に多数

```css
color: var(--primary);        /* #4F46E5 が入る */
background: var(--bg);        /* #F9FAFB が入る */
border-radius: var(--radius); /* 16px が入る */
```

---

## 12. CSS疑似クラス・疑似要素

### `:hover`
- **意味**: マウスを乗せたとき（ホバー時）に適用されるスタイル
- **用途**: リンクの色変化・カードが浮き上がるエフェクトなど
- **ポートフォリオでの使用箇所**: ナビリンク・スキルカード・ワークカード

```css
.skill-card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 24px rgba(0,0,0,0.08);
}
```

---

### `::before` / `::after`
- **意味**: 要素の前・後ろに仮想の要素（コンテンツ）を追加する
- **用途**: HTMLを触らずCSSだけで装飾を追加できる。必ず `content: ''` が必要
- **ポートフォリオでの使用箇所**: ヒーローの装飾円・contactカードの装飾

```css
.hero::before {
  content: '';          /* 必須。空でもOK */
  position: absolute;   /* 好きな場所に配置 */
  width: 300px;
  height: 300px;
  background: #C7D2FE;
  border-radius: 50%;
}
```

---

### `:nth-child(n)`
- **意味**: 親要素の中でn番目の要素にスタイルを当てる
- **用途**: 複数のカードを順番に少しずつ遅らせてアニメーションさせるなど
- **ポートフォリオでの使用箇所**: `.fade-up:nth-child(2)` など

```css
.fade-up:nth-child(2) { transition-delay: 0.1s; }
.fade-up:nth-child(3) { transition-delay: 0.2s; }
```

---

### `* { ... }`
- **意味**: ページ内のすべての要素に適用するCSSセレクター
- **用途**: `margin: 0; padding: 0;` でブラウザのデフォルト余白をリセットする
- **ポートフォリオでの使用箇所**: CSSの最初のリセット

---

### `p + p`
- **意味**: `<p>` の直後にある `<p>` だけに適用する（隣接セレクター）
- **用途**: 段落と段落の間だけに `margin-top` をつけるなど細かい制御に使う
- **ポートフォリオでの使用箇所**: `.about-card p + p { margin-top: 12px; }`

---

## 13. JavaScript：DOM操作

### `document.querySelectorAll('セレクター')`
- **意味**: ページ内でCSSセレクターに一致するすべての要素を取得する
- **用途**: 複数のクラスを持つ要素をまとめて操作する
- **ポートフォリオでの使用箇所**: `.fade-up` クラスを持つ要素を全部取得

```javascript
document.querySelectorAll('.fade-up') // .fade-up全部を取得
```

---

### `.classList.add('クラス名')`
- **意味**: 要素にクラスを追加する
- **用途**: JSでクラスを追加することでCSSのスタイルを動的に適用する
- **ポートフォリオでの使用箇所**: スクロールで `.visible` クラスを追加してアニメーション発動

```javascript
entry.target.classList.add('visible')
// 要素に class="visible" が追加される
// → CSSの .fade-up.visible のスタイルが適用される
```

---

### `.forEach(callback)`
- **意味**: 配列や要素リストの各要素に対して順番に処理を実行する
- **用途**: 複数の要素に同じ処理（監視・クラス追加など）をまとめて適用する
- **ポートフォリオでの使用箇所**: `.fade-up` の全要素を監視対象に追加

```javascript
document.querySelectorAll('.fade-up').forEach(el => {
  observer.observe(el) // 各要素を監視
})
```

---

## 14. JavaScript：イベント・監視

### `new IntersectionObserver(callback, options)`
- **意味**: 要素が画面（ビューポート）に入ったり出たりしたときに処理を実行する監視機能
- **用途**: スクロールで要素が見えたらアニメーションを発動させるなど
- **ポートフォリオでの使用箇所**: スクロール時のfade-upアニメーション

```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {   // 画面に入ったら
      entry.target.classList.add('visible') // クラスを追加
    }
  })
}, { threshold: 0.15 }) // 15%見えたら発動
```

---

### `observer.observe(要素)`
- **意味**: 指定した要素をIntersectionObserverの監視対象に追加する
- **用途**: どの要素をスクロール監視するかを登録する
- **ポートフォリオでの使用箇所**: `.fade-up` 全要素を監視対象に

---

### `entry.isIntersecting`
- **意味**: 要素が現在画面に見えているかどうかを表す `true` / `false` の値
- **用途**: `if (entry.isIntersecting)` で「画面に入ったとき」だけ処理を実行
- **ポートフォリオでの使用箇所**: スクロールアニメーションの条件分岐

---

### `threshold: 0.15`
- **意味**: 要素の何割（0〜1）が見えたらコールバックを呼ぶかの設定
- **用途**: `0.15` = 15%見えたら発動。`0` にすると少しでも見えたら発動
- **ポートフォリオでの使用箇所**: IntersectionObserverのオプション

---

## 15. JavaScript：制御構文

### `const`
- **意味**: 再代入できない変数を宣言する
- **用途**: 変わらない値・オブジェクトを格納するときに使う。基本的に `const` を使えばOK

```javascript
const observer = new IntersectionObserver(...)
```

---

### `if (条件) { ... }`
- **意味**: 条件が `true` のときだけ中の処理を実行する
- **用途**: 「画面に入ったときだけ」「クリックされたときだけ」など条件付き処理

```javascript
if (entry.isIntersecting) {
  // 画面に入ったときだけここが実行される
}
```

---

### アロー関数 `(引数) => { 処理 }`
- **意味**: 関数の短い書き方
- **用途**: `forEach` や `callback` に関数を渡すときによく使う

```javascript
// 通常の書き方
function(entry) { ... }

// アロー関数（同じ意味）
(entry) => { ... }

// 1行なら {} も省略できる
el => observer.observe(el)
```

# 逆引き：やりたいことから探す

## デザイン・見た目

| やりたいこと | 使うキーワード |
|---|---|
| 要素を横並びにしたい | `display: flex` |
| 格子状（2列・3列）に並べたい | `display: grid` + `grid-template-columns` |
| 要素を画面中央に置きたい | `margin: 0 auto` または `justify-content: center` |
| 上下中央に揃えたい | `align-items: center` |
| 角を丸くしたい | `border-radius` |
| 枠線をつけたい | `border` |
| 影をつけたい | `box-shadow` |
| 背景を半透明にしたい | `rgba(...)` / `opacity` |
| グラデーションをつけたい | `linear-gradient(...)` |
| 要素をはみ出さないようにしたい | `overflow: hidden` |
| 要素を重ねたい | `position: absolute` + `z-index` |
| 画面上部に固定したい | `position: sticky` |

## テキスト・フォント

| やりたいこと | 使うキーワード |
|---|---|
| 文字を中央揃えにしたい | `text-align: center` |
| リンクの下線を消したい | `text-decoration: none` |
| 文字を太くしたい | `font-weight: 700` |
| 文字サイズを変えたい | `font-size` |
| 文字の色を変えたい | `color` |
| 文字間隔を広げたい | `letter-spacing` |
| 行間を広げたい | `line-height` |
| 文字を大文字にしたい | `text-transform: uppercase` |
| リストの点（●）を消したい | `list-style: none` |

## リンク・ナビ

| やりたいこと | 使うキーワード |
|---|---|
| 別ページへリンクしたい | `<a href="URL">` |
| 同じページ内でジャンプしたい | `<a href="#id名">` + `id="..."` |
| メールを開くリンクを作りたい | `<a href="mailto:メールアドレス">` |
| スムーズにスクロールしたい | `scroll-behavior: smooth` |

## アニメーション・動き

| やりたいこと | 使うキーワード |
|---|---|
| ホバーで色が変わるようにしたい | `:hover` + `transition` |
| ホバーで浮き上がるようにしたい | `:hover` + `transform: translateY(-2px)` + `transition` |
| ページロード時にふわっと出したい | `@keyframes` + `animation` |
| スクロールしたら要素を出したい | `IntersectionObserver` + `classList.add` |
| アニメーションを時間差で出したい | `animation-delay` |

## レスポンシブ（スマホ対応）

| やりたいこと | 使うキーワード |
|---|---|
| スマホでだけCSSを変えたい | `@media (max-width: 480px)` |
| 2列をスマホで1列にしたい | `@media` 内で `grid-template-columns: 1fr` |

## 構造・管理

| やりたいこと | 使うキーワード |
|---|---|
| 色を一元管理したい | `:root` + `var(--変数名)` |
| 要素に名前をつけてCSSを当てたい | `class="..."` + `.クラス名` |
| 特定の1つの要素を指定したい | `id="..."` + `#id名` |
| CSS/JSから要素を取得したい | `document.querySelectorAll` |
| 複数要素に同じ処理をしたい | `.forEach()` |
| 条件によって処理を変えたい | `if (...)` |
